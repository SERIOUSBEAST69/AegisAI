1. 引言

1.1 编写目的

本文档旨在明确“可信AI数据治理与隐私合规工作台”的软件需求，为后续的系统设计、开发实现、测试验收提供依据。本文档的预期读者包括：项目开发团队、测试团队、项目管理者和评审专家。

1.2 项目背景

随着2026年新修《网络安全法》的正式施行，人工智能首次被纳入国家网络安全体系，明确要求企业对AI系统进行全生命周期风险监测、算法伦理审查和数据合规管理。同时，国家数据局《关于培育数据流通服务机构的意见》明确提出要培育数据商和数据治理服务机构。

然而，校园和中小企业普遍缺乏专业的IT安全团队，无法有效应对日益严格的合规要求。大厂提供的AI产品（如文心一言、通义千问等）本身是合规的，但当用户将自己的数据输入AI时，存在“产品合规 ≠ 使用合规”的巨大鸿沟。本项目旨在填补这一空白，为资源有限的用户提供轻量级、可负担的数据治理与隐私合规解决方案。

1.3 术语与定义

术语 定义
PII 个人身份信息（Personally Identifiable Information）
DPA 数据处理协议（Data Processing Agreement）
RBAC 基于角色的访问控制
ABAC 基于属性的访问控制
数据血缘 数据的来源、流向和被使用情况的追踪记录
动态脱敏 在数据查询时实时进行脱敏处理，不改变原始存储
被遗忘权 数据主体要求删除其个人数据的权利

2. 项目概述

2.1 产品定位

可信AI数据治理与隐私合规工作台是一个面向校园和中小企业的轻量级SaaS/私有化部署平台。它作为“AI使用的合规性中间件”，位于用户的数据和外部的AI服务之间，确保所有交互行为都合规、可控、可追溯。

核心价值主张：让没有专业安全团队的用户也能安全、合规地使用AI。

2.2 目标用户

· 校园用户：信息化中心管理人员、科研团队、行政人员、普通学生
· 中小企业用户：管理层、IT运维人员、合规岗、业务部门员工

2.3 产品范围与边界

包含范围：

· 数据资产的发现、分类、分级和血缘追踪
· AI模型的注册、风险标注和使用授权
· 细粒度的权限管控和审批流程
· 全链路操作审计和可视化追溯
· 合规策略的动态配置和实时拦截
· 异常行为检测和风险预警
· 自动化合规协议生成和数据脱敏
· 数据主体权利响应（查询/下载/删除）

不包含范围：

· 不开发自有的AI基础大模型
· 不替代专业级的数据治理平台（如大型数据中台）
· 不提供法律咨询服务（仅提供合规工具）

2.4 假设与约束

假设条件：

· 用户已有基础IT设施（服务器或云资源）
· 用户愿意配合完成初期的数据资产梳理
· 校园和中小企业接受SaaS订阅或轻量级私有化部署

约束条件：

· 必须遵循2026年《网络安全法》的合规要求
· 必须遵循GB/T 35273个人信息安全规范
· 开发语言以Java为主，便于维护和部署
· 如使用AI工具，必须在大赛指定的15款国产免费AI工具范围内选择

4. 功能需求

4.1 数据资产管理

功能描述：管理组织内的所有数据资产，进行分类分级和血缘追踪。

需求编号 功能点 详细描述 优先级
FR-101 数据源接入 支持MySQL、Excel文件、API等多种数据源接入；支持自动扫描敏感字段 P0
FR-102 敏感数据识别 内置NLP模型自动识别PII（姓名、身份证、手机号等）、财务数据、科研数据 P0
FR-103 分类分级 内置分类模板（公开/内部/敏感/受限），支持自定义标签；支持手动/自动打标 P0
FR-104 数据预览 在不暴露原始敏感数据的前提下，预览数据样本（敏感字段自动模糊） P1
FR-105 数据血缘 记录数据来源和流向，可视化展示被哪些AI模型调用过 P1
FR-106 批量导入 支持Excel/CSV批量导入数据资产清单 P2

4.2 AI模型管理

功能描述：管理组织内可使用的AI模型和服务，进行风险标注和使用授权。

需求编号 功能点 详细描述 优先级
FR-201 模型注册 支持接入外部API（配置密钥、端点）、上传内部模型（如ONNX/PMML格式） P0
FR-202 风险标签 设置模型风险等级（低/中/高），进行伦理预检（偏见/歧视风险评估） P1
FR-203 调用授权 配置可见范围（哪些角色/部门可用）、调用频次限制、每日配额 P0
FR-204 调用监控 展示近7天调用趋势、TOP调用用户、异常调用记录 P1
FR-205 模型下架 支持模型的停用、下架和历史数据归档 P2

4.3 权限与访问控制

功能描述：实现细粒度的权限管理和审批流程。

需求编号 功能点 详细描述 优先级
FR-301 用户管理 账号维护、角色分配、部门管理；支持单点登录集成（校园/企业） P0
FR-302 权限策略 支持基于角色的访问控制和基于属性的访问控制 P0
FR-303 数据级权限 支持行级权限（仅可见特定部门数据）、列级权限（隐藏敏感字段） P1
FR-304 审批流程 高风险访问需提交申请；支持多级审批（主管→合规官）；审批待办通知 P0
FR-305 紧急授权 支持临时授权通道，需合规官二次确认 P2

4.4 审计追溯中心（核心P0模块）

功能描述：记录所有操作行为，提供可视化审计和追溯能力。

需求编号 功能点 详细描述 优先级
FR-401 日志采集 记录谁 + 时间 + IP + 设备 + 操作类型 + 访问对象 + 输入输出概要（脱敏后） P0
FR-402 日志存储 结构化存储，至少保留6个月；防篡改设计（哈希链或数字签名） P0
FR-403 审计查询 支持多维度检索（用户/时间/资产/操作类型）；支持审计轨迹时序图 P0
FR-404 报表导出 一键生成合规报告PDF，支持监管报送格式 P1
FR-405 异常标注 审计员可对异常操作进行标记和备注 P1
FR-406 日志清理 支持日志归档和自动清理策略配置 P2

4.5 合规策略引擎

功能描述：配置和管理数据访问及AI调用的合规规则。

需求编号 功能点 详细描述 优先级
FR-501 内置规则库 预置网安法、个保法等常用规则模板 P1
FR-502 自定义规则 支持IF-THEN形式的规则配置（如：IF 角色=实习生 AND 操作=导出 AND 数据敏感等级=高 THEN 阻断） P1
FR-503 实时拦截 检测到违规访问时实时阻断请求，触发二次认证，并发送告警 P1
FR-504 策略生效范围 支持全局生效、指定数据资产生效、指定模型生效 P1

FR-602 预警方式 站内消息、邮件、钉钉/企微机器人、大屏红色告警 P1
FR-603 处置闭环 标记风险 → 调查 → 关闭，记录完整处置日志 P1
FR-604 风险看板 展示风险趋势、高风险事件TOP、处置效率 P2

4.7 合规自动化工具

功能描述：提供隐私合规相关的自动化能力。

需求编号 功能点 详细描述 优先级

FR-702 数据脱敏 支持动态脱敏（查询时实时处理）和静态脱敏（导出/备份时处理）；算法支持替换/掩码/加密 P2
FR-703 主体权利响应 支持数据主体查询个人数据、申请删除（被遗忘权）、数据导出（可携权） P2

模型资产 5-50个
访问日志 日均1000-10000条

· 日志数据：热存储3个月，冷归档6个月，过期自动清理
· 敏感数据：加密存储，备份数据同样加密
· 审计数据：防篡改保护，重要日志永久保留（如监管要求）
